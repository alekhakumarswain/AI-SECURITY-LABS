# Security Policy

## Supported Versions

Currently, the main branch of `AI Security Labs` is the only supported version for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

Security is a core focus of this project, both in its educational content and implementation. If you discover a security vulnerability within the platform itself (not the intentional vulnerabilities meant for learning in the labs), please do not disclose it publicly.

Instead, please report it via the following process:

1. **Email:** Send a detailed report to the repository owner or open an issue on GitHub with the title `[SECURITY BUG]`, providing as much context as possible.
2. **Details to include:**
   - A clear description of the vulnerability.
   - Steps to reproduce the issue.
   - Any potential impact or proof of concept (PoC).

### Response Timeline
We aim to acknowledge receipt of vulnerability reports within 48 hours and strive to resolve confirmed issues as quickly as possible. Upon resolution, a patch will be deployed and, if applicable, a public disclosure will be made acknowledging your contribution (unless you request to remain anonymous).

## Bug Bounty
At this time, we do not have an official bug bounty program, but we deeply appreciate the responsible disclosure of any security flaws.
